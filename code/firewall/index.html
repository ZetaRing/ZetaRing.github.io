Background & Obstacle

1. The firewall rules in the datacenter are not unified. There are some invalid firewall rules. And some teams will casually deploy their firewall rules.

2. There are more than one million firewall rules. Manually monitoring is impractical.

3. No one understands all firewall rules in the datacenter. It is hard to label all the firewall rules.

4. Hackers will change the firewall rules in several machines, making monitoring such behavior challenging since so many rules exist.

5. Firewall rules are dynamic. Applications will change firewall rules now and then.

Target

We need to design a method to detect abnormal firewall rules effectively.

Solution

I proposed two assumptions for this scenario. These two assumptions significantly reduce the problem's complexity, leading to a practical solution.

- Assumption

1. Most firewall rules are configured reasonably. (Most people are good people :)

2. A group of firewall rules should serve the same purpose. A single firewall rule can not satisfy the requirement of service. (People work in different groups)

- Deduction

Here is the critical point.

1. Firewall rules serving the same purpose should have the same spatial distribution in the datacenter.

2. If a firewall rule is confirmed to be configured reasonably, another firewall rule that shares the same spatial distribution should also be considered as configured reasonably.

- Conclusion

We can use an unsupervised learning algorithm to automatically cluster firewall rules into groups based on their spatial distribution. After that, labeling the group is enough.

Algorithm & Result

- Data representation

Each firewall rule will be represented by a vector that describes its spatial distribution.

Each element in the vector represents how many machines have this firewall rule in this location.

(Machines' physical group defines a location. Each location has more than one machine.)

- Clustering

By calculating the similarity of vectors, firewall rules can be clustered unsupervised.

Result of the similarity matrix. Yellow Triangles in the diagonal correspond to standard firewall rules. The red circle in the right picture represents outliers.

- Result

I did not use the names of firewall rules in the algorithm. But firewall rules with similar names can be discovered by the algorithm. This proves that 1) standard firewall rules can be discovered in the form of groups. 2) abnormal firewall rules can be detected since they will be outliers.
